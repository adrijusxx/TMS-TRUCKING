## Migration Principles

1. **Universal Table**: All data grids must use `@components/universal-table`
2. **Safety Gate**: Driver dispatch requires valid CDL & Medical Card (FMCSA compliance)
3. **Financial Integrity**: No direct pay edits - use LedgerEntry system (Deductions/Bonuses)
4. **MC Number Filtering**: All entities must support MC Number filtering
5. **US Geography**: All location logic must account for State Codes, Zip Codes, Timezones
6. **Units**: Miles (not km), Lbs (not kg), USD ($)
7. **No Duplication**: Reuse logic in `@src/lib` and UI in `@src/components`
8. **Straight Line Workflow**: Order -> Dispatch -> Active Load -> Delivery -> Invoice -> Factoring/Payment

## Notes

- **Complexity Levels**:
  - **Low**: Simple CRUD, minimal business logic
  - **Med**: Moderate business logic, some integrations
  - **High**: Complex workflows, multiple dependencies, integrations

- **Priority Order**: Complete Tier 1 before Tier 2, Tier 2 before Tier 3, etc.

- **Dependencies**:
  - MC Numbers must be implemented before most other features
  - Safety features (CDL, Medical Cards) must be implemented before Dispatch
  - Drivers, Trucks, Customers must be implemented before Loads
  - Loads must be implemented before Invoices and Settlements
